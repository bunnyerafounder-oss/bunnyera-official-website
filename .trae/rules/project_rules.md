# BunnyEra 项目规则

在本项目中，Trae 对话与代码生成必须遵循以下规则：

## 🧠 输出语言
- 默认使用 **中文** 输出，必要时附带英文解释。
- 技术文档统一使用 **Markdown** 格式，结构清晰，标题分级。

## 💻 代码生成
- 所有代码必须包含 **详细注释**，解释逻辑与用途。
- 命名规范：统一使用 **小写 + 下划线**，避免缩写和模糊命名。
- 默认生成可运行的完整示例，不允许省略关键部分。

## 🎨 设计与文档
- 所有文档必须简洁、可执行，不允许模糊描述。
- 所有视觉示例必须符合 BunnyEra 品牌色彩与风格。
- 响应式设计与模块化架构为默认要求。

## 🔄 项目流程
- 所有操作必须有明确步骤与说明，禁止临时决策。
- 所有变更必须记录在 README 或 changelog 中。
- 所有资源必须归档并命名规范。

## 🤝 沟通习惯
- 技术表达简洁直接，情感表达优雅深刻。
- 跨语言内容需保持语义一致，必要时优化为诗意表达。
- 回答必须完整、可执行，不允许模糊或省略。
